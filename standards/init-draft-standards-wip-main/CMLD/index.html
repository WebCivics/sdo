The Context Markup Language (CML) binds verbatim source text to machine-readable concepts and the logic that governs them; Context Markup Language Definitions (CMLD) maps surface terms onto those concepts. Both are realised natively on the QualiaDB engine and support Human-Centric AI.

Introduction

Software agents, particularly LLMs, struggle to disambiguate terms with multiple meanings ("bank" the institution vs. the riverbank; "physician" the person vs. the practice). Existing semantic-web forms (RDFa, JSON-LD) are author-heavy and not optimised for agent inference or for Human-Centric AI [[EU-AI-ACT]]. CML provides declarative, machine-readable context so meaning is stated, not guessed.

An earlier draft of CML proposed inline <cml:context> HTML elements and an external .ctx JSON file (CMLD). The implemented system keeps that purpose but realises it as a concept-graph layer native to the engine: the inline-element and sidecar-file mechanisms are superseded by the concept-as-context-hash model defined here.

Namespace

The CML namespace (the term identifiers) is https://ns.webcivics.net/cml/ (prefix cml:); the published schema-definition ("sdo") documents resolve at https://sdo.webcivics.org/. The authoritative machine axioms are published as core-ontologies/cml.n3. CML composes — rather than re-invents — established vocabularies: SKOS [[SKOS]], Web Annotation [[Web-Annotation]], PROV-O [[PROV-O]], ODRL [[ODRL]], OWL-Time, GeoSPARQL, and the Security Vocabulary; plus the WebCivics values: human-rights agent lattice.

The concept-as-context-hash model

CML's load-bearing idea: a concept is a context hash. A concept IRI is hashed (q_hash) into the 56-bit context field (bits [0..55]) of the Super-Quin (Vector 4). The concept's sub-graph is every quin whose context equals that hash. This is the engine's native named-graph mechanism — a document is a query for all quins intersecting a context hash — so logic libraries (deontic, temporal, defeasible) run against a concept by masking on its context hash, without loading the whole graph.

cml:Concept is rdfs:subClassOf skos:Concept and is modelled in RDFS + SHACL, never OWL, for the given world (see the modeling discipline).

The three-layer model

Nanopublication-aligned: an assertion graph (LOGIC) + a provenance graph (the realization) + pub-info (the curation signature).

  • TEXT — verbatim originalText, immutable (legal fidelity), stored cold in the .q42 front-of-file lexicon and recoverable from the archive alone.
  • CONCEPT — a context hash that owns a sub-graph; carries skos:prefLabel/skos:definition; related via SKOS. Linked to text by cml:realizedBy (a Web Annotation media-fragment selector + integrity hash). Many fragments — across instruments, languages, amended text — may realise one concept.
  • LOGIC — per-concept modality sub-graphs evaluated by the deterministic Sentinel VM, masked on the concept's context hash. The deontic sub-graph compiles to native operators (Obligate / Permit / Forbid) with q42:unless defeaters.

The Curation Prime Directive

Automated systems MAY assert only skos:closeMatch or skos:related. Only a cryptographically-signed human action MAY assert skos:exactMatch. Concept identity is curation-grade: auto-asserting sameness from vector proximity would re-create the algorithmic flattening this system exists to defeat. The machine proposes (cml:Proposed, with a confidence); a human disposes (cml:Attested, with a sec: signature). This is enforced closed-world by a SHACL [[SHACL]] shape (cml:AttestedExactMatchShape): a machine-written exactMatch without a ratifying attestation is a conduct violation. 

cml:AttestedExactMatchShape a sh:NodeShape ;
    sh:targetSubjectsOf skos:exactMatch ;
    sh:property [ sh:path cml:curationStatus ; sh:hasValue cml:Attested ; sh:minCount 1 ;
                  sh:message "skos:exactMatch requires cml:Attested (a signed authoritative human action)." ] .

CMLD — Context Markup Language Definitions

CMLD maps the surface terms people write onto concepts. A CMLD definition is a binding term → context DID, where the context DID denotes a CML concept (its IRI hashes to the concept's context hash). A "Context Markup Document" is the set of CMLD definitions that give a body of content its declared meanings, layered over the immutable TEXT.

  • Authoring: upsert_cmld_definition(term, context_did) records that term resolves to context_did.
  • Resolution: on ingest, CML/CMLD URIs are resolved against the dynamic Ontology Registry; the term's context_did selects the concept's sub-graph, so the agent reads the intended meaning and the logic bound to it.
  • One hash-space: because the definition's context_did is the concept's context hash, a CMLD definition joins directly into the concept graph — no separate index, no duplication.

CMLD definitions are CML-governed: a binding is cml:Proposed when machine-suggested and cml:Attested only on a signed human action; a sameness binding (skos:exactMatch) is subject to the Curation Prime Directive above.

Implementation status: the registry API and ingest-time resolution exist as the engine surface; full persistence and the document-ingest path are being wired. This section specifies the design the implementation embodies, not a claim of end-to-end completeness.

Modeling discipline — the OWL boundary & SHACL firewall

  • World of man (constructed) — legal persons, institutions, treaties, contracts, software agents, artifacts — MAY use OWL.
  • Natural world, including humans (given) — natural persons, ecology, observations — MUST use RDFS + SHACL, never OWL. A person is not an owl:Thing to be classified into existence.
  • External mappings carry reference, not ontological commitment — a skos:closeMatch to schema:Physician adopts the identifier for interop/disambiguation but never inherits its class axioms (never Physician ⊑ Place). The boundary is a bidirectional firewall.
  • Two enforcement layers compose — the Deontic + Temporal VM answers is this norm binding now? (Active / Defeated / Expired); SHACL answers does reality comply?, and a shape is applicable only while its norm is Active. The VM adjudicates validity; SHACL enforces compliance.

Serialization, layered outputs & the engine

  • SOURCE / interchange: RDF / N3 / Turtle (the cml.n3 axioms and concepts/*.n3).
  • Wire: CBOR-LD [[CBOR-LD]] is the primary transmission serialization (one hash-space; @context expansion).
  • Execution: the unified .q42 archive — header → lexicon → block index → block-dir → LZ4 40 KB SuperBlocks — preserving a two-step range fetch (fetch the small index; one HTTP Range for the single block).

Per source instrument the corpus emits three layers:

  1. *.n3 — foundational N3/Turtle + N3Logic rules + SHACL shapes (SOURCE, human-curated).
  2. *.cml.html — the CML-annotated human-readable rendering (GENERATED).
  3. *.q42 — the engine-native compiled volume (GENERATED).

The generated concept layer is reproducible from SOURCE at a stamped cml:schemaVersion; regeneration rewrites only the machine-cml:Proposed layer and never touches human cml:Attested attestations — the property that lets the logic→CML pipeline be re-run freely.

Examples

A concept with text, logic, and provenance

The implemented pilot — a treaty duty as a concept that owns a deontic sub-graph:

# TEXT (immutable, recovered from the .q42 lexicon)
<…convention-105#article-1> values:originalText
  "Each Member … undertakes to suppress and not to make use of any form of forced or compulsory labour …" .

# CONCEPT (context hash = q_hash("concept:DutyToSuppressForcedLabour"))
concept:DutyToSuppressForcedLabour
    a            cml:Concept ;
    skos:prefLabel "duty to suppress forced labour"@en ;
    cml:realizedBy [ a oa:Annotation ;
                     oa:hasTarget [ oa:hasSource <…convention-105#article-1> ;
                                    oa:hasSelector [ a oa:FragmentSelector ; rdf:value "char=0,87" ] ] ] ;
    cml:curationStatus cml:Proposed ;
    cml:asserts  [ a values:Obligation ; values:borneBy values:State ;
                   values:requires act:suppressForcedLabour ] .   # → OP_OBLIGATE in context Cdsf

Evaluating context Cdsf: an Active obligation borne by the ratifying party; an unless lawfully-authorised defeater in the same sub-graph flips it to Defeated.

A CMLD definition disambiguating a term

# upsert_cmld_definition("bank", "did:webizen:concept:FinancialInstitution")
# resolves, at ingest, to the concept whose context hash = q_hash(that concept IRI):
"bank"  skos:closeMatch  concept:FinancialInstitution ;   # machine-proposed
        cml:curationStatus cml:Proposed .
# a riverbank document would bind "bank" → concept:Riverbank instead.

Human-Centric AI alignment

  • Accurate inference — declared concepts replace probabilistic guessing.
  • Transparency / explainability — an agent can cite the concept, its realizing text span, and its provenance.
  • Fairness / inclusivity — multi-lingual realizations and curated concepts reduce dominant-meaning bias; the man-made/natural boundary refuses to redefine the given world.
  • Trust / accountability — the Curation Prime Directive makes sameness a signed human act, not a machine assertion.

Security & privacy considerations

  • Agents MUST validate URIs and DIDs before resolution [[DID-CORE]].
  • Authors SHOULD avoid sensitive data in concept labels/definitions; personal data is modelled via credentials over a fabric of identifiers, not as public literals.
  • Sameness assertions (skos:exactMatch) MUST be human-attested with a sec: proof (the Prime Directive), preventing silent machine flattening.

Conformance requirements use [[RFC2119]] terminology. A conforming implementation MUST realise concepts as context hashes, honour the three-layer separation (immutable TEXT), and enforce the Curation Prime Directive (machine closeMatch; human-attested exactMatch) via the SHACL firewall.

QualiaDB engine implementation note

  • Serialization: RDF / RDF-star at the model layer; CBOR-LD on the wire; a 48-byte binary Super-Quin in the .q42 archive for execution.
  • Logic & constraints: a native N3 streaming parser; explicit deontic operators (Obligate / Permit / Forbid) with q42:unless defeaters; SHACL for compliance.
  • Conflict & state: CRDT (Lamport-clock LWW) and paraconsistent routing manage temporally-bound state and contradictions (Allen interval / LTL) without system-wide failure.
